Illustrative Example: Touchalytics Project

This project asks students to build an Android authentication application that uses swipe behavior as a biometric signal, stores interaction data in cloud storage, and processes it with a Python-based machine learning backend.

Project Attributes

Greenfield vs Brownfield Project
Greenfield mobile application project with Firebase data storage and a Python-based machine learning backend.
Greenfield
Scope / Duration
Medium-to-large project, typically 6-8 weeks with staged milestones.
Multi-week
Technical Complexity
High. Requires mobile development, behavioral data capture, cloud storage, and machine-learning-based authentication.
High
Domain / Discipline
Mobile Computing, Behavioral Biometrics, and Machine Learning
Interdisciplinary

Competency Statement
Design a mobile behavioral-authentication application that captures swipe interactions, stores user data, and applies machine learning to verify identity.

Instructor Support Package

The Touchalytics instructor materials help instructors adopt the project, plan its deployment, guide teams through the mobile-cloud-ML workflow, and record maintenance guidance as tools and platforms change.

Project Overview
A written and video project overview introduces the project package, project attributes, learning objectives, task structure, task scaffolding, and guidance for when and how to use the project. The overview also describes possible variations, such as changing the programming language, cloud provider, or biometric signal.
INS1
Sample Solutions
A working solution codebase and video demonstrations show expected outcomes for each project phase, including swipe capture, enrollment, Firebase data storage, backend prediction, verification, and final authentication behavior.
INS4
Instructor Notes
Deployment notes document the project file structure, four project tasks, support resources, sample grading, common student challenges, team check-ins, setup support, Git use, Android gesture detection, Firebase integration, backend development, machine learning preparation, end-of-project reflection, and materials that may need updates when Python versions, machine learning libraries, Android Studio, Android SDKs, Firebase settings, or setup workflows change.
INS2
Reference Course
A sample syllabus and schedule show how Touchalytics has been integrated into a course, including lectures, setup labs, Android swipe capture, Firebase storage, Python backend development, machine learning, and final app-backend prediction integration.
INS3

Student Support Package

The Touchalytics student materials help teams set up the required mobile, cloud, and backend environments, build prerequisite knowledge, and complete the project phases for swipe-based mobile authentication.

Task Scaffolding
Task scaffolding is organized around Android app development, Firebase cloud database integration, Python backend development, and machine learning integration. It provides varied levels of scaffolding, from step-by-step guidance to more basic task requirements.
STU2
Installation Guidelines
Written and video guides help students set up Python 3 with an IDE, pip, and machine learning libraries; Android Studio for mobile development; and Firebase with Realtime Database configuration.
STU1
Laboratory Exercises
Dedicated setup labs walk students through Android Studio, Python backend setup, and Firebase configuration in class, with clear deliverables such as a Hello World Android app, a basic backend workflow, and verified database connectivity.
STU3
Reference Materials
Slides introduce behavioral biometrics and refresh Python machine learning with libraries such as scikit-learn. Reference links include videos, machine learning library documentation, Android swipe-event extraction resources, Android and Firebase documentation, example artifacts, and project report templates.
STU4

Assessment Package

The Touchalytics assessment materials organize evaluation around the project's four numbered implementation phases, while also helping instructors assess transfer beyond the original mobile authentication scenario.

Phased Milestones
Four scaffolded milestones build on one another with increasing difficulty: Task 1, build a swipe-enabled Android app; Task 2, extract and save swipe data to Firebase; Task 3, implement a Python backend for touch behavioral biometrics; and Task 4, integrate swipe data with the Python backend for prediction.
ASM1
Transfer-oriented Design Problems
The package includes ten transfer-oriented design problems that ask students to apply project ideas to new contexts and scenarios. It also includes a structured instructor prompt for generating additional design problems with an LLM.
ASM2
Detailed Rubric
A detailed rubric evaluates each project phase with suggested grading weights and criteria for functional implementation, data capture and storage, backend integration, model behavior, prediction accuracy, and final demonstration quality.
ASM3

Why This Project Design Achieves FORAP Design Goals

Task Alignment with Learning Objectives
The four project tasks translate the learning objectives into a staged implementation sequence: building a mobile authentication interface, capturing swipe interactions, storing behavioral data in Firebase, processing the data in a Python backend, and applying machine learning to verify identity.
GOAL1
Support Materials Aligned with Tasks and Objectives
Student slides, setup labs, support guides, references, milestones, and rubrics are aligned with the project tasks and learning objectives so students receive targeted support for Android development, swipe-data capture, Firebase integration, Python backend development, machine learning, and teamwork.
GOAL1 GOAL3
Modular Task and Support Structure
The project is organized around phased tasks with corresponding support materials for each phase, allowing instructors to reuse or adapt the Android, Firebase, Python backend, machine learning, and integration portions of the package independently.
GOAL2 GOAL3
Flexible Technical Design
The project design is flexible enough for instructors to change the programming language, backend framework, cloud provider, database, machine learning library, or biometric signal while preserving the phased task structure and learning objectives.
GOAL2
Task Shielding Through Black-boxing
Instructors can shield selected tasks by treating them as black boxes when they are outside students' preparation or course objectives. For example, an Android-focused course can provide the machine learning code, while a Python or machine learning course can provide a working Android app.
GOAL2 GOAL4
Teaching and Learning Support
Written and video setup guides, labs, background slides, references, README materials, instructor notes, sample schedules, and solution demos support instructor deployment and help students make progress through common technical and project-task challenges.
GOAL3 GOAL4
Scaffolded Challenge
The project remains high-complexity, but its challenge is made appropriate through phased tasks and task scaffolding, including task instructions, milestone checklists, required configuration files, troubleshooting guidance, and solution hints.
GOAL3 GOAL4
Formative Assessment
Setup labs, staged milestones, weekly check-ins, and intermediate deliverables give instructors clear points for feedback before students complete the final integrated system.
GOAL5
Summative and Transfer Assessment
The final app behavior, backend prediction, demonstration, presentation, detailed rubric, and transfer-oriented design problems support summative evaluation of implementation quality and assessment of learning transfer beyond the original project scenario.
GOAL5
