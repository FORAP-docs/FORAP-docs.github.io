Project Attributes

In addition to its support and assessment components, each FORAP project is described through a set of attributes that help instructors understand the nature, fit, and adaptability of the project. These attributes act as a concise project profile, helping instructors compare projects, judge course fit, and identify where adaptation may be needed for different contexts, constraints, and instructional goals.

Greenfield vs Brownfield Project
Whether the project starts from a new codebase students build from scratch or an existing system students modify, extend, test, refactor, or maintain.
ATT1
Scope / Duration
The expected size of the project and the time required for completion, from shorter modules to full-semester projects.
ATT2
Technical Complexity
The expected level of difficulty in terms of technical depth, integration effort, tools, concepts, and implementation demands.
ATT3
Domains / Disciplines
The computing areas or subject contexts in which the project is situated such as software engineering, cybersecurity, databases, artificial intelligence, or web development.
ATT4
Competency Specification Optional
Learning objectives can be written in a clear, structured, and comparable format so they can be compared across projects, courses, and curricula, such as through the competency model of CC2020.
