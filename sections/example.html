Illustrative Example: OAuth Project Package

This example shows how FORAP can package a computing project for reuse and adaptation. The OAuth project is organized as a phased team project in which students build a basic authentication subsystem, integrate external OAuth providers, and then create a custom OAuth server or identity provider.

Three-Task Structure

In this package, the three project tasks also function as phased milestones for teamwork and formative assessment. Each phase introduces a more advanced level of OAuth implementation while preserving a clear progression from foundation to integration to full provider design.

Task 1
Build a basic web authentication subsystem as the project foundation, including the initial client-side and server-side flow needed for secure sign-in.
Milestone 1
Task 2
Integrate third-party OAuth providers so students work with real external identity services and handle provider configuration, redirects, and token exchange.
Milestone 2
Task 3
Create a custom OAuth server or identity provider, requiring students to move beyond consumption of an existing service and design core provider behavior themselves.
Milestone 3

Package Components In This Example

FORAP packages the OAuth project through coordinated student, instructor, and assessment support. Across all three components, materials are available in both text and video formats to improve accessibility and reuse.

Student Support Package
Includes background materials on single sign-on, federated identity, and OAuth; startup lab exercises such as environment setup and connecting to an existing OAuth 2.0 provider; and task-specific guidance used directly to complete the three project tasks.
Student-facing
Instructor Support Package
Includes notes on structuring and deploying the project, sample deployment schedules, guidance on team size, timing, grading, project variations, and sample solutions aligned with the three tasks.
Instructor-facing
Assessment Package
Includes the three task phases as assessment checkpoints together with a grading rubric aligned to the expected student work across the project.
Assessment-facing

Why This Example Works For FORAP

Reusable Structure
The project decomposes naturally into three phases, making it easy to reuse as-is or adapt to a shorter or longer course deployment without losing instructional coherence.
Layered Difficulty
Students progress from foundational authentication concepts to integration work and then to provider design, which supports scaffolded learning in a technically demanding area.
Package Alignment
Student support, instructor support, and assessment checkpoints all map directly onto the same three-task project flow, which keeps the package internally consistent.
Assessment Visibility
The phased structure gives instructors clear formative checkpoints for feedback, grading, and team progress monitoring before the final project completion stage.
