Illustrative Example: Touchalytics Project

This project asks students to build an Android authentication application that uses swipe behavior as a biometric signal, stores interaction data in cloud storage, and processes it with a Python-based machine learning backend.

Assessment Package

The Touchalytics assessment materials organize evaluation around the project's four numbered implementation phases, while also helping instructors assess transfer beyond the original mobile authentication scenario.

Phased Milestones
Four scaffolded milestones build on one another with increasing difficulty: Task 1, build a swipe-enabled Android app; Task 2, extract and save swipe data to Firebase; Task 3, implement a Python backend for touch behavioral biometrics; and Task 4, integrate swipe data with the Python backend for prediction.

Why This Project Design Achieves FORAP Design Goals

Competency-to-task Alignment
The four project tasks directly support the competency statement by connecting mobile authentication, behavioral swipe data collection, Firebase storage, Python backend processing, and machine learning classification.
GOAL1
Outcome-to-material Alignment
Student slides, setup labs, support guides, references, milestones, and rubrics map to the intended outcomes in behavioral biometrics, Android development, Firebase integration, Python backend development, machine learning, and teamwork.
GOAL1
Reusable Package Structure
The project can be reused because it is organized into project tasks, instructor notes, sample solutions, implementation examples, installation guides, labs, support materials, and assessment rubrics.
GOAL2
Technology and Biometric Adaptation
Instructors can adapt the programming language, backend framework, cloud provider, database, machine learning library, or biometric signal while preserving the same project architecture.
GOAL2
Task Shielding With Black-box Components
Instructors can focus students on selected tasks and provide black-box solutions for others. An Android-focused course can provide the ML backend as runnable code, while a Python or ML course can provide a built Android app for testing integration and prediction.
GOAL2
Instructor and Student Support
Written and video setup guides, labs, background slides, references, README materials, instructor notes, sample schedules, and solution demos reduce setup barriers and adoption effort.
GOAL3
Scaffolded Challenge
The project remains high-complexity, but the work is staged from Android UI and swipe capture to Firebase storage, Python backend development, machine learning classification, and final prediction integration.
GOAL4
Formative Assessment
Setup labs, staged milestones, weekly check-ins, and intermediate deliverables give instructors clear points for feedback before students complete the final integrated system.
GOAL5
Summative and Transfer Assessment
The final app behavior, backend prediction, demonstration, presentation, detailed rubric, and transfer-oriented design problems support evaluation of implementation quality and learning transfer.
