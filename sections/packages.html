Support Packages

FORAP organizes each project as a coordinated package built around aligned learning objectives and supported through three complementary components: an Instructor Support Package, a Student Support Package, and an Assessment Package. Together, these components are intended to support reuse and adaptation, reduce adoption effort, support student progress, and strengthen project assessment.

Instructor Support Package

The Instructor Support Package includes the materials needed to help instructors adopt, implement, and sustain a project with less effort. It is intended to reduce common barriers related to time, planning, project organization, student support, and long-term sustainment.

Project Overview
A summary of the project’s purpose, objectives, competency specification, domains/disciplines, scope/duration, technical complexity, greenfield vs brownfield project type, and nature of tasks, along with guidance for adopting or adapting the project to a course context, including alignment with course goals, expected use, and local implementation considerations.
INS1
Instructor Notes
Notes that support project implementation and adaptation, including past deployment experiences, sample schedules, alternative project designs, common student challenges, and guidance for instructional decisions during project use. These notes also identify files, dependencies, tools, setup instructions, and services that may need updates as technologies change.
INS2
Reference Course
Examples of past project deployments in courses that help instructors understand how the project has been integrated into the syllabus, course schedule, assessment, and classroom use in practice.
INS3
Sample Solutions
Example solutions for project tasks similar to those expected from students, which can support instructor preparation, demonstration, and evaluation.
INS4

Student Support Package

The Student Support Package includes the materials provided to help students make progress during project work. It is designed to address common technical challenges students face, especially installation/configuration issues, prerequisite gaps, and difficulties encountered while completing project tasks.

Installation Guidelines
Materials that help students install, configure, and access the required tools and environments for the project, such as compilers, IDEs, DBMS, Web servers, and existing systems. These may include written or video guides, as well as setup artifacts such as virtual machines, containers, and installation scripts that automate or package parts of the setup.
STU1
Task Scaffolding
Project-specific scaffolding that students use directly to complete tasks, such as task instructions, milestone checklists, starter code, required configuration files, task-specific troubleshooting guidance, solution hints, diagrams, and templates. The level of scaffolding may vary from step-by-step direction to more basic task requirements.
STU2
Laboratory Exercises
Targeted exercises that help students get started with the project and build prerequisite knowledge and prepare for project tasks, such as learning to use a programming or markup language, GitHub, a database system, or a language framework.
STU3
Reference Materials
Resources students consult as needed for background understanding, starting with lecture materials and slide decks for key prerequisite knowledge. They may also include external references such as official documentation for APIs, languages, and frameworks, protocol specifications, report templates, example artifacts, and other supporting references.
STU4

Assessment Package

The Assessment Package includes materials used to support formative and summative assessment in a FORAP project. It is organized around the project's learning objectives so that assessment remains connected to what students are expected to know and be able to do.

Phased Milestones
Structured milestones and checkpoints that define intermediate deliverables and enable formative assessment during project execution, including feedback points tied to milestone artifacts.
ASM1
Transfer-oriented Design Problems
A set of scenario-based problems that assess learning beyond the original project tasks and can be deployed in exams, presentations, or project reviews to elicit higher-order thinking aligned with Bloom's Taxonomy.
ASM2
Detailed Rubric
A grading rubric aligned with the learning objectives, with clear criteria for evaluating project artifacts such as code, design documents, reports, and presentations.
