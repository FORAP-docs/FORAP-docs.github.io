Project Attributes

In addition to its support and assessment components, each FORAP project is described through a set of attributes that help instructors understand the nature, fit, and adaptability of the project.

Competency Specification
Learning objectives should be written in a clear, structured, and comparable format so they can be compared across projects, courses, and curricula, such as through the competency model of CC2020.
ATT1
Domain / Discipline
The computing area or subject context in which the project is situated.
ATT2
Scope / Duration
The expected size of the project and the time required for completion, from shorter modules to full-semester projects.
ATT3
Technical Complexity
The expected level of difficulty in terms of technical depth, integration effort, tools, concepts, and implementation demands.
ATT4
Codebase Context
The kind of codebase involved in the project, including greenfield projects built from scratch and brownfield projects that modify an existing system.
