Purpose of FORAP

The purpose of FORAP is to address key needs that affect the design, adoption, implementation, and sustainment of computing PjBL projects. It responds to recurring challenges faced by instructors and students and to the need for stronger project assessment support.

Instructor Imperative
  • Reduce the effort required to design new projects or adapt existing ones
  • Support alignment of projects with course learning objectives
  • Support project scoping, organization, and classroom implementation
  • Reduce the burden of preparing student support and assessment materials
  • Support long-term project sustainment and reuse across offerings
Student Imperative
  • Address installation and configuration challenges
  • Address prerequisite knowledge gaps
  • Support students while completing project tasks such as debugging, implementation, testing, deployment, version control, and documentation
  • Reduce technical barriers that can weaken student motivation, confidence, and progress
  • Provide coordinated support that helps students keep moving forward during project work
Assessment Imperative
  • Support formative and summative assessment within project work
  • Keep assessment aligned with the project's learning objectives
  • Make student progress visible through milestones and checkpoints
  • Support evaluation of project artifacts through clear rubrics
  • Support higher-order thinking assessment beyond the original project tasks through transfer-oriented design problems
